04

스프린트 정책 — Shape Up

각 스프린트를 설계 → 개발 → QA 세 기간으로 나눠 관리. 막대(타임라인)는 설계 기간만 표시, 개발·QA는 별도 트랙.

■ 설계 기간

= 분석 + 기획 + 디자인. 설계자(기획자겸 디자이너) 담당. 종료 시점이 개발 핸드오프.

■ 개발 기간

= 빌드 + 개발 AC 정합. 개발자 담당.
걸리는 기간(주)은 추정치 ⚠️ 개발팀 확정 필요.

■ QA 기간

환경 테스트 → 스테이지 → 프로드.
① 개발자 테스트 서버 QA
② 설계자 스테이지 확인
③ 통과 시 프로드 배포

진행 원칙

  • 설계 선행 → 개발 후행. 핸드오프 직전 1~2주 겹침 — 확정된 부분부터 개발 착수.
  • SDS는 별도 스프린트 아님 — 전 구간 상시 병행 (7월 시드 → 상시 확장).
  • 브랜딩은 시각적 완결성 위해 7월 초 별도 진행(공통).
